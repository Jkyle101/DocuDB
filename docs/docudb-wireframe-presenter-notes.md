# DocuDB Wireframe Presenter Notes

## File to open
- docs/docudb-uiux-wireframe.html

## Suggested talk sequence (7 minutes)
1. Show **System Architecture and Role Access**.
2. Explain **User Workspace** wireframes: Drive, Shared/Recent, Groups, Forms, Settings.
3. Explain **COPC** wireframes: Dashboard, Upload, Department Review, QA Review, Evaluation.
4. Explain **Superadmin** wireframes: Admin Files, Manage Users, Groups, Logs, Trash, COPC Admin.
5. End with **Mobile Wireframes** and **End to End UX Flows**.

## Key points to emphasize
- Role-based access prevents unauthorized actions.
- COPC flow is stage-driven and traceable from upload to evaluation.
- Admin modules support governance: user lifecycle, logs, archive, and compliance controls.
- Desktop and mobile structures remain consistent for easier onboarding.
