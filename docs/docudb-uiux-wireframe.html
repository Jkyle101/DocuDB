  1. Start with role matrix and navigation architecture.
  2. Show user side: My Drive, Shared, Groups, Forms, Settings.

What this deck proves

  1. Role-based access is visible in UI structure.
  2. Main modules follow a consistent navigation pattern.
  3. COPC review states are traceable across stages.
  4. Admin governance is separated from user operations.
  5. Layouts adapt to mobile for key tasks.

End to End UX Flows

Flow A: Faculty to QA

1: Faculty uploads documents in COPC Upload.
2: Department Chair reviews against checklist.
3: Rejected files return with revision notes.
4: Approved files proceed to QA review.
5: QA verifies and advances to evaluation.

Flow B: Superadmin Governance

1: Initialize COPC program and assign roles.
2: Monitor tasks, logs, and compliance health.
3: Resolve user or group access issues.
4: Compile package and finalize COPC-ready state.
5: Archive program and preserve audit trail.

Talking points for defense

  1. Architecture first: role-aware navigation and controlled permissions.
  2. Operational core: drive-like document handling and collaboration.
  3. Compliance core: COPC stage-by-stage decision checkpoints.
  4. Governance core: superadmin observability and lifecycle control.

Extra note

The wireframe intentionally uses low-fidelity boxes so reviewers focus on structure, workflow clarity, and role behavior, not visual polish.
